1. The real tokens are on a different chain and cannot be bridged cheaply.

3. The rating is the important part; the underlying token is a stand-in.

**The ratings attached to these mock addresses reflect the archetype of the real credit**, not the state of any particular bridged instance. Do not confuse the two.

## How to update this file

After running `script/Deploy.s.sol` and `script/SeedRatings.s.sol`:

1. Open the broadcast log under `broadcast/Deploy.s.sol/84532/run-latest.json` ŌĆö this has the core addresses
2. Open `broadcast/SeedRatings.s.sol/84532/run-latest.json` ŌĆö this has the MCC-* addresses in order
3. Fill in the table columns above
4. Commit (the `.github/workflows/snapshot.yml` cron will then refresh `docs/api/v0.4.1/ratings.json` nightly)

Alternative: run `tools/snapshot.py` manually, which parses the table below and calls `cast call` once per row.
