## 2. Token budget

### 2.1 Per-project input size

A single evidence-pack prompt is approximately:

- System prompt: ~180 tokens
- Rubric bundle (index.json + 7 per-dim files): ~8500 tokens
- Evidence pack (typical): ~800 tokens
- Output schema reminder: ~400 tokens
- **Total input: ~9,900 tokens**

Output per credit: 7 scores + disqualifier list + adjustment list + 7 one-sentence reasonings ≈ **700 tokens**.

| Platform | Hardware | $/hr | Tokens/sec (batched) | Cost per 2000-credit run (both models) |
|----------|----------|-----:|---------------------:|---------------------------------------:|
| Together.ai serverless | — | (pay per token, priced above) | — | ~$122 (listed above) |

## 3. Staging plan

### Stage 1: 168-project MVP (Claude-only) — $43

Run now once `ANTHROPIC_API_KEY` is provisioned.

```bash
export ANTHROPIC_API_KEY=sk-ant-...
python3 data/llm-panel-scale/run_panel.py --run-id bct168-mvp --providers claude-only --only-real
python3 data/llm-panel-scale/run_panel.py --run-id bct168-stress \
    --packs-dir data/llm-panel-scale/stress-packs --providers claude-only --only-real
python3 data/llm-panel-scale/validate.py --run-id bct168-mvp --stress-run-id bct168-stress
```

Expected runtime: 30-60 minutes (three Claude models in parallel, ~5s per call × 168 × 3).

### Stage 2: 500-project stratified expansion (Claude-only) — $128

Add 332 non-BCT projects stratified across:
- Methodology (VM0007, VM0009, VM0015, VM0044, VM0047, AMS, CDM-legacy, Puro, Isometric)
- Vintage (pre-2016, 2017-2020, 2021-2024, 2025+)
- Type (renewable, REDD+, ARR, IFM, DAC, mineralization, biochar, industrial gas, cookstove)

Sample list generated by `tools/sample_500.py` (to be written); packs built with `--online` Playwright scraper once that is added.

### Stage 3: 2000-project full target (claude-only) — $614

All VCS + GS + Puro + Isometric + CAR projects with at least one issuance in 2020-2026. Pack-build time: ~8-12 hours with rate-limited online scraping; panel-run time: ~4-6 hours.

### Stage 4: Add GPT-5 + Gemini + open-weight (full panel on 2000) — $1,150 total

Rerun the panel with the extended provider set. The runner is idempotent — rows already scored by Claude are not rescored.

## 5. Bill-of-materials

1. **Compute** (if running open-weight locally, not via Together):
   - 1x RunPod 8xH100 / 8xH200 node, on-demand, $18-28/hr
   - Disk: 5 TB SSD for Llama-4-405B + DeepSeek-R2 weights (~$50/mo)

2. **API credentials**:
   - Anthropic API key with >= 200M token quota
   - OpenAI API key with GPT-5 access
   - Google AI Studio key with Gemini 2.5 Pro
   - Together.ai API key (if using hosted OSS path)

3. **Human budget** (not API):
   - 20 Delphi experts × 50 credits × 2 rounds × 15 min/credit × $150/hr ≈ **$7,500** for the expert subset (Paper 1 §6)
   - Optional: 1 postdoc at 20% for 3 months on scale-up ops ≈ **$6,000**

## 6. Risk register

- **Rate limits.** Anthropic concurrent-request limits are typically 50-100 for research tier. The runner sends calls sequentially per process; parallelizing requires a 5-worker pool (trivially added; see `run_panel.py` TODO).
- **JSON parse failures.** The runner logs these as `mode: "parse-error"` and does NOT fill with synthetic values. Re-running the runner picks them up again idempotently.
- **Prompt drift.** If the rubric changes between runs, scores become non-comparable. The runner pins rubric_bundle version inside each row's `provenance` field to detect this.
- **Vendor model deprecation.** The v0.6 panel is explicitly multi-provider to limit single-vendor risk.
- **Evidence-pack staleness.** PDDs get re-issued; we rebuild packs quarterly at a minimum.

## 7. Decision: when to trigger each stage

- **Stage 1 (MVP)** is unblocked as soon as `ANTHROPIC_API_KEY` is set.
- **Stage 2 (500)** requires the Playwright scraper (to pull real Verra page content beyond the metadata fallback).
- **Stage 3 (2000)** requires Stage 2 to have shipped and the panel to have demonstrated Fleiss' κ > 0.4 on the 500-project run.
- **Stage 4 (full panel)** requires GPT-5 and Gemini keys and one of {Together.ai credits, dedicated compute}.

Submission target: Stage 1+2 numbers in the NeurIPS 2027 submission (November 2026 deadline); Stage 3+4 in the camera-ready and/or KDD 2027 submission.
